1. 
Params - вписок всех параметров, учавствующих в мониторах
ParamTypes - типы параметров, для их группировки при выборе или отображении на экране. (не более того)

Параметры могут быть объединены в пакеты (Packets/PacketsParams), в это случае, при формировании наборов параметров для монитора, см. п.2, параметры одного пакета 
вставляются однвременно, нвзависимо от того, какой параметр из пакета выбран. Т.е. данные паарметры имеею смысл все вместе, а не по отдельности. 
Например, пакет показаний тонометраЖ: Верх.давление, нижнее давление, частота пулься.

2.Каждый параметр имеет свой тип данных ParamValueTypes и единицу измерения Units. Типы данных параметра: Целое, Вещественное, Текст. 
При выборе текста параметр НЕ используется в построении графиков. Для ежедневных записей типа текст можно реализовать простую записную книжку - ежедневник
Для параметров вещественных типов указывается количество знаков после запятой, можно ограничить 0,1,2,3,4,5,6 - обсуждается, поле DECIMAL(28,8)- точность выбрать по-умолчанию
Для текстовых значений можно ввести два типа : строка - varchar(128)(комментарий)  и текст varchar(512) - обсуждается -> ограничение по объему памяти
Все значения параметров в зависимости от типа  фиксируются в MonitoringParamNumericValue или MonitoringParamStringValue(?) - к обсуждению 

3.
Monitors - наборы параметров для фиксации значений по датам. Это типовые шаблоны, на основании которых формируются параметры ежедневной фиксации измерений - мониторинги,
Monitorings.

Добавил: 
поля в таблицу параметров монитора (MonitorParams):
- св-во вычисляемое поле(Calculated), если да, то значение формируется на основе связанных параметров dbo.MonitorParamLinks.
- тип действия над параметром (таблица ParamActions): 0 - стандартное 1 - сложение(накопление), т.е. при установке значения оно добавляется к текущему. 2- вычитание и т.д.

таблицу dbo.MonitorParamLinks, в этой таблицы тип действия означает как обрабатываются значения параметров, складываются, вычитаются , умножаются и т.д.
Пример: пусть новый параметр Остаток:
1)делаем его вычисляемым, например баланс между приходом и расходом, в  dbo.MonitorParamLinks два параметра Приход и Расход, первый со сложением, второй с вычитанием.(тип действия 1 и 2)
2)для этого параметра устнваливаем тип действия : сложение. Т.о. получаем остаток ДС на любую дату  

Итак, есть:
1. Структура для формирования динамики изменения данных.
2. Если параметр нечисловой, то можно реализовывать простой ежедневник, см. п.2 

Чтобы хотелось:
Есть идея, использовать такую группу монитора, как финансы - приход/расход. Но в этом случае, к простому отражению динамики статей движения дс, нужен расчет баланса.
Т.е. по-видимому, будет расширение свойств параметра монитора.Что пока не предусмотрено, т.к. в рамках текущей модели - простая динамика числовых параметров. Это к обсуждению.
